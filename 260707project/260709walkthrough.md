# 🚀 LangGraph 기반 Multi-Agent 동화 생성 파이프라인 구축 완료

## 1. 개요 및 목적

기존의 단일 LLM(Monolithic) 프롬프트 방식에서 발생하던 **치명적인 할루시네이션(쌍따옴표 사용, JSON 포맷 파괴 등)**을 근본적으로 해결하기 위해, 각 역할을 분담하는 **멀티 에이전트(Multi-Agent) 아키텍처**를 도입했습니다.

## 2. 주요 변경 사항 (LangGraph Architecture)

> [!TIP]
> 엔터프라이즈 레벨의 안정성을 확보하기 위해 파이프라인을 5개의 개별 노드(Node)로 쪼개어 단일 책임 원칙(SRP)을 부여했습니다.

1. **Retrieve Node**: `kb_retriever.py`를 호출하여 아이의 성향과 문제 상황에 딱 맞는 레퍼런스(Chroma DB)를 검색합니다.
2. **Draft Node**: 시스템 프롬프트(나이, 성별, 제약조건)를 바탕으로 한국어 동화 텍스트 초안만 집중해서 생성합니다.
3. **Review Node (핵심 방어벽)**: 생성된 초안에 치명적 오류(쌍따옴표 `"` 등)가 있는지 스스로 검수합니다. 오류 발견 시 즉각 **반려(Loop)**하여 LLM이 스스로 고쳐오도록(Self-Correction) 강제합니다.
4. **Translate Node**: 다중 언어 지원! 입력된 `language` 값이 `en`일 경우에만 영미권 동화 스타일로 완벽하게 번역합니다.
5. **Format Node**: LangChain의 `PydanticOutputParser`를 사용해 프론트엔드가 요구하는 완벽한 JSON 스키마 규격으로 변환합니다. `cover_image_prompt`와 `image_prompt`를 영어로 일관되게 작성하도록 통제합니다.

## 3. 테스트 및 성능 튜닝 결과

### ✅ 1) Self-Correction 루프 작동 증명

테스트 결과, LLM이 습관적으로 대화체에 쌍따옴표를 넣으려고 시도했으나 `Review Node`를 통한 검수로 최종적으로 작은따옴표만 사용한 안전한 텍스트를 만들어 냅니다.

### ⚠️ 2) 이미지 병렬 생성 튜닝 및 롤백 (무료 API 한계)

- **도입 시도**: 1분이 넘게 걸리던 이미지 대기 시간을 줄이기 위해 `ThreadPoolExecutor`를 도입하여 5장의 그림 생성을 동시에(Parallel) 쏘도록 비동기 처리했습니다.
- **결과 및 롤백**: Pollinations API 측에서 엄격한 IP당 Rate Limit(초당 요청 수 제한)을 걸어두어 `429 Too Many Requests`가 무더기로 발생했습니다. (5장 중 2장만 성공).
- **조치**: 무료 인프라 환경의 한계를 인정하고, 안정성을 위해 원래의 **순차적(Sequential) 5초 딜레이 생성 코드**로 깔끔하게 롤백을 완료했습니다. 추후 DALL-E 같은 유료 API 연동 시 병렬 처리 코드로 손쉽게 복구할 수 있습니다.

## 4. 최종 결과물 요약

> [!NOTE]
> 한국어 버전과 영어 버전 모두 완벽한 기-승-전-결과 **캐릭터 디자인 일관성(Image Prompt 고정)**을 확보했습니다.

- **스토리 퀄리티**: 아이의 행동과 심리를 잘 나타내는 구어체, 의성어/의태어 활용 및 "현실 식탁에서의 냅킨 실천"과 같은 아동 심리 전문가 수준의 감정선.
- **JSON 안정성**: LLM이 본문에 억지로 쌍따옴표를 덧붙이더라도 Pydantic Parser가 완벽하게 이스케이프(`\"`) 처리하여 프론트엔드 연동이 무너지는 것을 100% 방지.
