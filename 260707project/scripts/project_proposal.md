# [초기 기획서] AI 맞춤형 행동 교정 동화 서비스

## 1. 서비스 개요

- **서비스 목적**: 아이의 나쁜 습관이나 사회성 부족 문제를 해결하기 위해, 아이가 몰입할 수 있는 초개인화된 맞춤형 행동 교정 동화를 생성하는 서비스.
- **핵심 타겟**:
  - **사용자 (생성 주체)**: 3~5세 유아동을 둔 부모님
  - **최종 독자 (소비 주체)**: 3~5세 유아동

## 2. 사용자 입력 명세 (Input Data)

부모님이 아이에게 딱 맞는 동화를 생성하기 위해 입력하는 설정값입니다.

### 2.1. 기본 인적 사항
- **이름**: [텍스트 입력란] (예: 민준, 지우)
- **나이**: [드롭다운] (3세 / 4세 / 5세)
- **성별**: [체크박스] (남자 / 여자 / 선택 안함)

### 2.2. 아이의 성향 및 테마 (구조화된 템플릿 입력)
- 서술형 입력의 막막함을 방지하기 위해 가이드라인과 템플릿을 제공.
- **성격 키워드**: [체크박스/태그 선택 - 최대 3개] (예: 장난스러움, 호기심 많음, 부끄럼쟁이 등)
- **선호 테마**: [드롭다운] (예: 마법 세계, 우주 탐험, 동물 친구들)

### 2.3. 교정이 필요한 문제 상황 (예시 및 자동완성 제공)
- **문제 카테고리**: [드롭다운] (일상 습관 / 식습관 / 사회성 등)
- **상세 상황 (1~2줄)**: [텍스트 입력란] 
  - *입력 팝업 가이드 제공*: "예시: 양치질을 싫어해서 칫솔만 보면 도망가요."

## 3. 동화 스토리텔링 구조 및 메커니즘

입력된 데이터를 바탕으로 AI는 다음과 같은 흐름으로 동화를 전개합니다.

1. **주인공 등장**: 아이의 이름과 성향을 그대로 가진 주인공 등장. (예: 공주를 좋아하는 5세 지우)
2. **갈등 발생**: 부모가 입력한 문제 상황이 동화적 판타지로 발현. (예: 양치를 안 해서 입속에 충치 괴물 왕국이 세워짐)
3. **모험과 깨달음**: 주인공이 조력자(요정 등)를 만나거나 문제를 극복하는 과정을 통해, 올바른 행동(양치질 하기)이 스스로를 구하는 용기 있는 행동임을 깨달음.
4. **긍정적 결말**: 올바른 행동 후 얻게 되는 긍정적인 보상(깨끗해진 치아 요정들의 감사 인사)으로 훈훈한 마무리.

## 4. AI 동화 생성 엔진 학습 데이터 (Training Data Requirements)

자연스럽고 교육적인 동화 텍스트를 생성하기 위해 LLM(예: Solar 등)에 주입해야 할 학습/프롬프트 데이터입니다.

- **유아동 타겟 문체 데이터 (Tone & Manner)**:
  - 3~6세 인지 수준에 맞춘 쉬운 어휘 사용.
  - 풍부한 의성어와 의태어 (예: 폴짝폴짝, 우물우물, 뽀득뽀득, 쿵쾅쿵쾅).
  - 아이에게 다정하게 읽어줄 수 있는 구연동화식 종결 어미 (예: "~했단다", "~했어요", "~지 뭐예요?").
- **전래/창작 동화 코퍼스 데이터**:
  - 한국적 정서와 윤리(효, 우애, 나눔)가 녹아있는 동화 플롯.
  - 기승전결의 깔끔한 이야기 전개 방식 학습.
- **아동 심리 및 행동 치료 가이드라인 (Safety & Ethics)**:
  - **긍정적 강화 원칙**: 공포·협박("양치 안 하면 도깨비가 잡아간다") 묘사를 원천 차단하고, 프롬프트에 '공포/협박 키워드 사용 금지' 로직을 명시.
  - **사회적 이야기(Social Stories) 구조 강제**: 상황 → 행동 → 결과 → 대안 → 보상의 5단계 구조를 프롬프트 템플릿에 반영.

## 5. 시스템 아키텍처 및 AI 모델링 전략

### 5.1. 텍스트 모델 파이프라인 (Solar LLM 기반)

- **대규모 데이터셋 및 RAG 모델 구축**: 글로벌 동화 데이터(Kaggle)와 한국어 코퍼스를 번역/정제하여 대규모의 고품질 동화 지식 베이스(Vector DB)를 구축함.
- **RAG(검색 증강 생성) 기반 동적 퓨샷(Dynamic Few-Shot)**:
  - 부모의 고민(입력값)과 가장 유사한 상황/배경의 **기존 동화 예시**를 실시간으로 검색해 프롬프트에 주입(Few-shot)함으로써, 스토리가 산으로 가는 것을 방지하고 문체의 퀄리티를 유지함.
- **합성 데이터 플라이휠 (Synthetic Data Flywheel)**:
  - 사용자가 생성한 동화 중 부모 피드백 별점이 높은(예: 4점 이상) 고품질 생성물을 다시 RAG 지식 베이스(임베딩 풀)에 자동으로 편입시킴.
  - 서비스가 운영될수록 도메인에 최적화된 독자적인 고품질 합성 데이터셋이 무한 증식하며, 다음 생성 품질을 끌어올리는 선순환(자가 발전) 생태계를 구축함.
- **분량 및 페이지 제어**: 3~4페이지 분량의 이야기를 안정적으로 생성하기 위해, 프롬프트에 '총 4페이지 구성', '각 페이지당 3~4문장', '기-승-전-결 플롯 할당'을 명시적으로 강제함.

### 5.2. 이미지 생성 파이프라인 (멀티모달 구성)

- **투트랙(Two-track) 아키텍처**: 텍스트 생성을 담당하는 Solar LLM과 이미지 생성을 담당하는 외부 모델(DALL-E 3 또는 Stable Diffusion)을 백엔드에서 엮어(Model Chaining) 사용.
- **작동 방식**:
  - Solar LLM이 페이지별 동화 텍스트뿐만 아니라, 해당 장면을 묘사하는 **영문 이미지 프롬프트(image_prompt)**를 JSON 형태로 함께 출력.
  - 백엔드가 이 프롬프트를 이미지 생성 API로 전송하여 이미지를 반환받음.
- **캐릭터 일관성(Consistency) 유지 전략**:
  - **프롬프트 제어 및 Seed 고정 (MVP)**: 주인공의 고정된 외모 묘사를 모든 페이지의 이미지 프롬프트에 강제로 이어 붙임. 또한, 이미지 API 호출 시 **동일한 Seed 값**을 적용하여 이미지 간의 편차를 최소화.
  - **이미지 캐싱 (MVP)**: 동일한 프롬프트/페이지가 중복 요청될 경우 캐시(Redis 등)를 활용하여 API 호출 비용 및 대기 시간 절약.
  - **SD + LoRA / IP-Adapter (고도화 단계)**: 아이의 실제 사진을 활용하여 얼굴을 고정시키는 Stable Diffusion 기술 도입을 고려하여 개인화 경험 극대화.

## 6. 데모 인프라 구축 및 비용 최적화 (Cost Optimization)

- **웹 서버 호스팅**: 백엔드 서버는 무거운 연산을 직접 하지 않으므로 AWS EC2 무료 티어(t2.micro) 또는 Vercel, Render 같은 무료 PaaS 플랫폼을 사용하여 유지비용을 0원으로 억제함.
- **AI 연산 처리 (텍스트 및 이미지)**:
  - **텍스트 (Solar LLM)**: 프로젝트 기간 동안 API가 제공되므로, 비용(토큰 수) 걱정 없이 RAG(동적 퓨샷)나 프롬프트 엔지니어링에 긴 예시 텍스트를 자유롭게 주입하여 동화의 퀄리티를 극한으로 끌어올림.
  - **이미지 (Stable Diffusion / DALL-E 3)**: 데모 단계에서 AWS 내부에 비싼 GPU 서버를 직접 구축하는 것을 피하고, 그림이 생성되는 순간(초 단위)에만 과금되는 **서버리스 GPU API(fal.ai, Replicate 등)**를 활용하여 이미지 연산 측의 요금 폭탄을 방어함.

## 7. 데이터 및 프라이버시 보호 (Data & Privacy)
- **부모 동의 및 암호화**: 가입 시 아동 데이터 수집 동의 절차를 마련하고, 민감한 개인정보(이름, 성향 등)는 안전하게 처리.
- **익명화**: 실제 인물 사진을 업로드하지 않는 MVP 단계에서는 철저히 텍스트 기반 익명 처리로 법적 위험(PIPA 등)을 회피함.

## 8. 향후 구현 로드맵 (MVP 중심)
1. **텍스트 MVP (1~2개월)**: 템플릿 기반 입력 UI 구현, Solar LLM + RAG(ChromaDB) 연동, 프롬프트 엔지니어링 및 시나리오 템플릿 DB 구축.
2. **이미지 MVP (2~3개월)**: 서버리스 이미지 API 연동, 고정 프롬프트 및 Seed 적용, 이미지 캐시 도입.
3. **피드백 시스템 (3~4개월)**: 행동 교정 효과 파악을 위한 부모 설문 추가, 품질 평가 데이터를 바탕으로 프롬프트 가중치 개선.
